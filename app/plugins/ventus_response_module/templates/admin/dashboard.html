    Vendor Dispatch Bridge

    One-way: when a callsign is allocated on CAD, Ventus POSTs an incident to the configured vendor endpoint. If that callsign has a terminal UUID in the map below, allocation uses terminal (direct handset). Otherwise allocation uses channel (default channel or per-callsign channel map). Master unlock is database-only. Inbound webhooks from the vendor are not used.

    Enable outbound only after keys are set: UPDATE ventus_broadnet_settings SET master_enabled = 1 WHERE id = 1;

    Channel UUID per callsign

    Optional. If a callsign has no terminal below, the vendor uses this channel (or the default channel field above).

    Handset (terminal) UUID per callsign

    Optional. When set, dispatch targets that handset instead of a channel for that callsign.
